End to end

How the in-browser tools actually work

No magic — just modern browsers. WebAssembly runs real document engines on your machine, so the file never leaves.

1

You drop a file

It's read straight into the browser tab's memory — never sent anywhere.

2

Your device does the work

WASM engines (pdf-lib, ffmpeg, codecs) process it locally, using your CPU.

3

You get the result

The output is built in-memory and downloaded — still nothing uploaded.

4

Nothing is left behind

Close the tab and it's gone. No account, no server copy, no trace.

The honest exception: a few tools genuinely need a server — Office conversions and OCR (LibreOffice/OCR can't run in a browser), and AI. For those we say so up front, send over an encrypted connection, and delete your file the moment it's converted — never stored, never read. Office conversions are 3 free a day; Pro removes the cap.
