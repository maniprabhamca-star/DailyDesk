Anki-compatible CSV export is the differentiator — students paste straight into Anki/Quizlet. Smallpdf's question generator has no flashcard export. Every question carries its page cite so teachers can verify against the source.
Privacy line (all three tools, verbatim from Chat with PDF): the file never leaves your device — pdf.js extracts the text in your browser and only text is sent, over TLS, to our server and on to the AI. Nothing is stored, nothing is logged as content, never used for training. This is stated in the UI exactly like /chat-pdf.

Build notes (shared plumbing — why batch 1 is fast)

  • 1One backend pattern: each tool = a new endpoint in routes/ai.js reusing the exact control stack Chat with PDF already has — availability check → Pro/owner gate → per-user daily cap + global USD kill-switch (aiBudget) → per-tool guard() kill-switch → Haiku call. No new infrastructure.
  • 2One frontend pattern: reuse lib/pdf-chat.ts extraction + the Chat-with-PDF left panel; only the right-hand controls/output differ per tool.
  • 3Gating: all three ship coming_soon → owner-only via ToolGate (same as /chat-pdf), flipped to pro at Pro launch. Public sees "coming soon" pages that already rank (title/desc/JSON-LD/sitemap from day one).
  • 4Cost (Haiku): Summarize ≈ $0.01/run · Questions ≈ $0.01 · Translate ≈ $0.01–0.15 (whole text in+out — hence the 30-page/run cap). All inside the existing $5/day global kill-switch.
