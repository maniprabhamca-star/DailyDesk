What we can and cannot see (shown in the UI, verbatim)

  • ✓We store: the sealed bytes, file size, and when it was uploaded.
  • ✗We cannot see: the contents — not us, not a breach of our servers, not a subpoena served on us. There is no key on our side.
  • ✗We cannot reset your passphrase. The recovery key is the only fallback — that's a feature, and the UI says so before you enable the vault.
  • ✓File names: encrypted too (the grid decrypts them locally after unlock).

Build plan (phased — each phase verified before the next)

  • 1Crypto core (on-device): Argon2id key derivation + AES-256-GCM sealing in a worker; unit-tested vectors; file names encrypted.
  • 2Storage backend: ciphertext-only chunked upload/download on the VPS (quota per plan), zero knowledge of keys.
  • 3Vault UI: the three screens above + folders, search-by-name (local), progress, and the "what we can't see" panel.
  • 4Sync + polish: multi-device unlock, conflict-safe versions, soft-delete recycle bin (client-side sealed).
